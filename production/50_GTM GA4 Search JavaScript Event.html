  • 페이지뷰 트리거 구현
    GTM에서 맞춤변수를 활용하여 검색어를 추적하는 방법 
    맞춤변수 + 상호작용할 GTM 세팅 안내
    Custom JavaScript Variable 구현
        세팅 : 좌측 Custom JavaScript Variable코드를 참고하여 검색어를 추적할 수 있는 JavaScript 코드를 구현 후 변수를 생성한다.
        작동방식 : Custom JavaScript Variable은 URL에서 사용자가 입력한 검색어를 추출하여 변수에 저장한다.
        GTM과 상호 작용 방식 : view_search_results 이벤트(GTM사용)를 GTM에 수동으로 보낸다. 해당 이벤트에는 search_term 매개변수도 포함된다.

    GA4 이벤트 태그 구현
        세팅 1 : 태그 생성 시, 태그 내 이벤트 이름은 정확히 view_search_results로 지정한다
        세팅 2 : 이벤트 매개변수는 search_term을 입력하며, 매개변수의 값은 위에서 생성한 Custom JavaScript Variable를 삽입한다
        작동방식 : 트리거로 인해 Fired된 GTM태그는 view_search_results 이벤트와 매개변수 searchTerm의 값(Custom JavaScript Variable)의 값을 GA4로 전달

    페이지뷰 트리거 구현
        세팅 1 : 위에서 생성 한 맞춤 변수를 세팅하며, 정규식 ^(undefined|null|0|false|NaN|)$ 을 조건으로 지정한다.
        조건식 의미 : cjs – 검색어 (사용자 정의 JS 변수)가 정규식과 일치하지 않는다(대소문자 무시)라는 의미를 내포
        작동방식 : 검색어를 입력한 페이지뷰가 발생하고 페이지 URL에 검색어가 포함된 경우 트리거가 실행
  • Custom JavaScript Variable 삽입 시 유의사항

  • Custom JavaScript Variable 구현 시 주의사항   /search/ or /search/results/

    검색어 앞에 URL의 페이지 경로가 어떻게 표시 되는지에 따라 예시코드 3행의 코드 값이 달라짐
        검색 URL이 yoursite.com/search/your-search-term 인 경우
        : Custom JS 변수에 /search/ 를 입력
        검색 URL이 yoursite.com/search/results/your-search-term/ 인 경우
        : Custom JS 변수에 /search/results/ 를 입력

    페이지 경로에 정의한 searchParam 이 포함되어 있지 않으면 해당 사용자 정의 JS 변수는 undefined 를 반환

  • 검색결과 동적 할당   Dynamic Value Allocation

    각 매개변수는 검색 완료페이지에서 해당하는 값이 할당되어야 함

    검색 이벤트 발생시, 각각의 검색 정보가 이벤트 내 매개변수에 동적으로 할당되도록 코드 구현 필요


  • 필수 매개변수   Required parameters

    매개변수는 몇 개를 선택하여 사용 가능하지만, 필수 매개변수는 반드시 구현되어야 함

    searchTerm은 필수 매개변수


  • dataLayer 개발 가이드 문서   Guide Document

    https://developers.google.com/tag-platform/tag-manager/templates/create-a-custom-variable


GTM - GA4 검색어 추적 구현 > Setting Process

Process 1

1. 변수 구성

변수 구성 단계

  • Process
    01
    변수 구성 
     1번 : [변수]를 클릭
     2번 : [새로 만들기]를 클릭
    변수 구성 
     3번 : 변수 이름을 지정. Custom JavaScript Variable이므로 cjs - search query로 이름 지정
     4번 : [변수 구성]을 클릭
  • Process
    02
    변수 구성 
     1번 : 변수 유형에서 [맞춤 자바스크립트]를 선택
    변수 구성 
     2번 : 상단 주의사항에 맞게 맞춤 자바스크립트 값을 입력.
     해당페이지는 yoursite.com/search/your-search-term 인 경우를 예시로 진행(/search/값 사용)
     3번 : [저장]버튼 클릭
  • Process
    03
    변수 구성 
     1번 : 저장이 되고 있는 모습
    변수 구성 
     2번 : 올바르게 세팅이 되었다면 저장 후 변수 목록에 노출 됨

Process 2

2. 태그 구성

태그 구성 단계

  • Process
    04
    태그 구성 
     1번 : 태그 탭 선택
     2번 : 새로만들기를 클릭하면 오른쪽에서 태그 생성창이 열림
    태그 구성 
     3번 : 태그 이름 작성. 구글에서 지정한 공식 이벤트 이름인 view_search_results로 지정. 해당 방법은 view_search_results를 추적하는 1번째 방법이므로 (1)을 부여
     4번 : [태그 구성]을 클릭하면 다시 한번 오른쪽에서 태그 목록이 열림
  • Process
    05
    태그 구성 
     1번 : Google 애널리틱스 : GA4 이벤트 선택
    태그 구성 
     2번 : 태그 유형에는 [Google 애널리틱스 : GA4 이벤트]가 제대로 선택됬는지 확인
     3번 : 구성 태그에는 '[GA4]측정 ID태그'를 선택(GA4의 측정ID를 불러오는 태그)
     4번 : 이벤트 이름에는 'view_search_results'를 입력(소문자로만). 구글에서 지정해준 이벤트 이름
     5번 : [이벤트 매개변수]를 클릭해서 추가 메뉴를 확장
     6번 : [매개변수 이름]에는 'search_term'를 입력(소문자로만). 구글에서 지정해준 매개변수 이름
     7번 : [값]에서 우측 '+' 버튼을 클릭하여 세팅창으로 이동
  • Process
    06
    태그 구성 
     1번 : 조금전 생성한 맞춤변수인 '[GA4] cjs - search query' 선택
    태그 구성 
     2번 : 상단 이미지와 같이 세팅되면 올바르게 진행된 것

Process 3

3. 트리거 구성

트리거 구성 단계

  • Process
    07
    트리거 구성 
     1번 : [트리거]를 클릭하면 오른족에 트리거 유형을 선택할 수 있는 창이 열림
    트리거 구성 
     2번 : 트리거 선택에서 [+]를 클릭
  • Process
    08
    트리거 구성 
     1번 : 트리거 이름을 입력. view_search_results 태그와 일관성을 유지하기 위해 [GA4] Search Trigger로 입력함
     단, 해당 방법은 Search Trigger를 추적하는 1번째 방법이므로 (1)을 부여
     2번 : [트리거 구성]을 클릭한다
    트리거 구성 
     3번 : 트리거 유형에서 [페이지뷰]를 선택
  • Process
    09
    트리거 구성 
     1번 : [페이지뷰]로 올바르게 세팅되었는지 확인
     2번 : 이 트리거는 [일부 페이지 뷰]에서만 실행되어야만 함
     3번 : 맞춤 변수인 [GA4] cjs- search query 선택
     4번 : 정규 표현식과 일치하지 않음(대소문자 무시) 선택
     5번 : 정규식 ^(undefined|null|0|false|NaN|)$ 입력
     6번 : [저장] 선택
     페이지뷰가 발생하고 페이지 URL에 검색어가 포함된 경우 이 트리거가 실행
    트리거 구성 
     7번 : 네모칸과 같이 세팅이 완료되었다면 정상적으로 진행 된 것
     7번 : [저장] 버튼을 선택하고 완료
  • Process
    10
    트리거 구성 
     1번 : 네모칸과 같이 세팅이 완료되었다면 정상적으로 진행 된 것

Process 4

4. 태그 최종 게시(웹사이트 배포)

작업 완료된 태그 최종 게시

  • Process
    11
    최종 게시 전 테스트 및 디버깅  
     조금 전 생성한 태그가 작업공간의 변경사항에 추가 된 것을 확인
     이상 없을 경우 미리보기 클릭
    최종 게시 전 테스트 및 디버깅  
     구글 태그 매니저 컨테이너 스크립트가 삽입된 홈페이지 URL을 삽입
     [connect]를 클릭하면 홈페이지가 열리면서 미리보기 모드와 연결 됨
  • Process
    12
    최종 게시 전 테스트 및 디버깅  
     연결이 완료되면 팝업창이 Connected로 내용이 변경 됨
     Continuned를 클릭하여 테스트 실행
    최종 게시 전 테스트 및 디버깅  
     미리보기 화면에서 [Tags Fired] 영역에 설정해둔 태그들이 보인다면 제대로 작동하는 것
     의도한 대로 조건에 맞게 태그가 Fired 되는지도 체크
  • Process
    13
    태그 배포  
     실제로 웹 사이트에 가상 태그를 배포하는 작업
     구글 태그 매니저 메인 화면에 접속하여 [제출]을 클릭
    태그 배포  
     1번 : 구글 태그 매니저는 협업으로 많이 활용하기에 해당 작업을 누가 했는지 표식을 남김. 대체로 날짜와 작업자 이름 기입
     2번 : 버전 설명에는 배포한 태그명을 작성. 또는 편의대로 설명을 추가해도 무방
     3번 : [게시]를 클릭하면 모든 과정이 완료되고 가상 태그가 웹 사이트에 구현
  • Process
    14
    태그 배포  
     "듣는 사람이 없는 것 처럼 노래를 부르세요" 라는 문구가 등장하면 태그가 올바르게 배포되고 있는 것
    태그 배포  
     배포가 완료되었으며 배포된 버전에 대한 설명 화면이 노출 됨
  • Process
    15
    태그 배포  
     1번 : [게시]한 후에는 작업공간의 변경사항이 초기화
     2번 : 방금 게시한 버전의 번호와 작업자 이메일, 게시 날짜 등이 표시 됨
    태그 배포  
     배포가 완료되었으며 배포된 태그는 태그 탭에 목록형으로 노출
