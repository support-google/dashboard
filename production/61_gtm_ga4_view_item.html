  • Key

  • 필수
  • 설명

  • name

  • 텍스트
  • *예
  • 제품의 이름 * ID 또는 name 중 하나를 설정해야 한다(예: P67890)

  • 제품과 연결된 브랜드(예: xexymix).

  • 제품이 속한 카테고리(예: 의류).구분 기호로/를 사용하여 계층 구조를 최대 5개 수준(예: 의류/남성/티셔츠)으로 지정

  • 정수

  • 정수
  • 아니오
  • 목록 또는 컬렉션에서 제품의 위치(예: 2)
  • 코드 삽입 시 유의사항

  • 거래별 동적 가치 할당   Dynamic Value Allocation

    각 매개변수는 결제 완료페이지에서 해당하는 값이 할당되어야 함

    거래발생시, 각각의 거래 정보가 이벤트 내 매개변수에 동적으로 할당되도록 코드 구현 필요


  • 단일 거래정보, 복수 제품정보   Single transaction information, multiple product information

    1번의 이벤트(view_item)에 제품이 여러 개인 경우, items코드는 여러번 호출되어야 함

    좌측 예시코드에 한글로 써 둔 결과값은 실제 거래 발생 시 거래에서 해당하는 정보를 가져와야 함


  • 필수 매개변수   Required parameters

    매개변수는 몇 개를 선택하여 사용 가능하지만, 필수 매개변수는 반드시 구현되어야 함

    id, name은 필수 매개변수


  • GTM - add_to_cart 개발 가이드 문서   Guide Document

    https://developers.google.com/analytics/devguides/collection/gtagjs/enhanced-ecommerce


GTM - GA4 view_item 이벤트 구현 > Setting Process

Process 1

1. 변수 구성

변수 구성 단계

  • Process
    01
    변수 구성 
     1번 : [변수]를 클릭
     2번 : [새로 만들기]를 클릭
    변수 구성 
     3번 : 변수 이름을 지정. 전자상거래 정보 중 제품정보는 Ecommerce.items를 많이 사용
     4번 : [변수 구성]을 클릭
  • Process
    02
    변수 구성 
     1번 : 변수 유형에서 [데이터 영역 변수]를 선택
    변수 구성 
     2번 : 데이터 영역 변수 이름에 [ecommerce.purchase.items]를 입력.
     3번 : [저장]을 클릭
  • Process
    03
    변수 구성 
     1번 : 위와같이 변수 목록에 추가되면 제대로 생성된 것이다

Process 2

2. 태그 구성

태그 구성 단계

  • Process
    04
    태그 구성 
     1번 : 태그 탭 선택
     2번 : 새로만들기를 클릭하면 오른쪽에서 태그 생성창이 열림
    태그 구성 
     3번 : 태그 이름 작성. 구글에서 지정한 공식 이벤트 이름인 'view_item' 로 지정.
     4번 : [태그 구성]을 클릭하면 다시 한번 오른쪽에서 태그 목록이 열림
  • Process
    05
    태그 구성 
     1번 : Google 애널리틱스 : GA4 이벤트 선택
    태그 구성 
     2번 : 태그 유형에는 '[Google 애널리틱스 : GA4 이벤트]'가 제대로 선택됬는지 확인
     3번 : 구성 태그에는 '[GA4]측정 ID태그'를 선택(GA4의 측정ID를 불러오는 태그)
     4번 : 이벤트 이름에는 'view_item'를 입력(소문자로만). 구글에서 지정해준 이벤트 이름
     5번 : [이벤트 매개변수]를 클릭해서 추가 메뉴를 확장
  • Process
    06
    태그 구성 
     1번 : [이벤트 매개변수]를 클릭해서 추가 메뉴를 확장
     2번 : 매개변수 이름에 첫 번째로 [items]를 입력
     3번 : [값]에서 우측 '+' 버튼을 클릭하여 세팅창으로 이동
    태그 구성 
     4번 : 미리 생성해둔 데이터 영역 변수 [GA4] Ecommerce.items를 선택
  • Process
    07
    태그 구성 
     1번 : 상단 이미지와 같이 세팅되면 올바르게 진행된 것

Process 3

3. 트리거 구성

트리거 구성 단계

  • Process
    06
    트리거 구성 
     1번 : [트리거]를 클릭하면 오른족에 트리거 유형을 선택할 수 있는 창이 열림
    트리거 구성 
     2번 : 트리거 선택에서 [+]를 클릭
  • Process
    07
    트리거 구성 
     1번 : 트리거 이름을 입력. 'view_item_list' 태그와 일관성을 유지하기 위해 [GA4] view_item Trigger로 입력함
     2번 : [트리거 구성]을 클릭한다
    트리거 구성 
     3번 : 트리거 유형에서 [맞춤 이벤트]를 선택
  • Process
    07
    트리거 구성 
     1번 : 트리거 유형이 [맞춤 이벤트]로 제대로 세팅되었는지 확인
     2번 : 이벤트 이름으로는 'view_item' 입력
     3번 : 이 트리거는 [모든 맞춤 이벤트]에만 실행되어야만 함
    트리거 구성 
     5번 : 네모칸과 같이 세팅이 완료되었다면 정상적으로 진행 된 것
     6번 : 저장 버튼을 선택하고 완료
  • Process
    07
    트리거 구성 
     1번 : 저장중 이라는 화면 뜸
    트리거 구성 
     2번 : 네모칸과 같이 세팅이 완료되었다면 정상적으로 진행 된 것

Process 4

4. 태그 최종 게시(웹사이트 배포)

작업 완료된 태그 최종 게시

  • Process
    08
    최종 게시 전 테스트 및 디버깅  
     조금 전 생성한 태그가 작업공간의 변경사항에 추가 된 것을 확인
     이상 없을 경우 미리보기 클릭
    최종 게시 전 테스트 및 디버깅  
     구글 태그 매니저 컨테이너 스크립트가 삽입된 홈페이지 URL을 삽입
     [connect]를 클릭하면 홈페이지가 열리면서 미리보기 모드와 연결 됨
  • Process
    09
    최종 게시 전 테스트 및 디버깅  
     연결이 완료되면 팝업창이 Connected로 내용이 변경 됨
     Continuned를 클릭하여 테스트 실행
    최종 게시 전 테스트 및 디버깅  
     미리보기 화면에서 [Tags Fired] 영역에 설정해둔 태그들이 보인다면 제대로 작동하는 것
     의도한 대로 조건에 맞게 태그가 Fired 되는지도 체크
  • Process
    10
    태그 배포  
     실제로 웹 사이트에 가상 태그를 배포하는 작업
     구글 태그 매니저 메인 화면에 접속하여 [제출]을 클릭
    태그 배포  
     1번 : 구글 태그 매니저는 협업으로 많이 활용하기에 해당 작업을 누가 했는지 표식을 남김. 대체로 날짜와 작업자 이름 기입
     2번 : 버전 설명에는 배포한 태그명을 작성. 또는 편의대로 설명을 추가해도 무방
     3번 : [게시]를 클릭하면 모든 과정이 완료되고 가상 태그가 웹 사이트에 구현
  • Process
    11
    태그 배포  
     "듣는 사람이 없는 것 처럼 노래를 부르세요" 라는 문구가 등장하면 태그가 올바르게 배포되고 있는 것
    태그 배포  
     배포가 완료되었으며 배포된 버전에 대한 설명 화면이 노출 됨
  • Process
    12
    태그 배포  
     1번 : [게시]한 후에는 작업공간의 변경사항이 초기화
     2번 : 방금 게시한 버전의 번호와 작업자 이메일, 게시 날짜 등이 표시 됨
    태그 배포  
     배포가 완료되었으며 배포된 태그는 태그 탭에 목록형으로 노출
