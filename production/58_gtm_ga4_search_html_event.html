    맞춤 HTML을 사용하는 추적 방식
    일부 웹사이트는 자동 완성 검색 기능을 사용한다.(검색 용어를 입력하기 시작하면 몇 초 후에 결과가 나타나기 시작)
    이를 GA4에서 추적하는 방식은 다음과 같다.
        방문자가 검색 필드에 검색어를 입력하기 시작한다.
        몇 초 동안 입력을 일시중지하거나 아예 중지하면, 가상 페이지뷰를 수정된 페이지 필드와 함께 GA4로 전송한다.

    GTM에서 사용자 정의 HTML을 활용하여 검색어를 추적하는 방법 
    사용자 정의 HTML + 상호작용할 GTM 세팅 안내
    사용자 정의 HTML 태그 구현
        세팅 : 좌측 맞춤 HTML을 참고하여 페이지의 자동 완성 검색을 추적할 수 있는 HTML태그를 구현한다.
        사용자 정의 HTML 태그가 활성화되면 해당 태그는 웹사이트의 자동 완성 검색 필드와 상호 작용을 하기 시작한다.
        이를 통해 검색 데이터가 데이터 영역으로 푸시 된다.
        해당 데이터를 사용하려면 데이터 영역 변수와 사용자 정의 이벤트 트리거 그리고 GA4 이벤트 태그를 생성해야한다.

    사용자 정의 HTML 트리거 생성(DOM)
        세팅 : DOM Ready 라는 새 트리거를 만든다.
        옵션1 : 검색 필드가 모든 페이지에 있는 경우 모든 DOM 사용 가능 이벤트에서 실행되도록 트리거를 설정
        옵션2 : 검색 필드가 특정 페이지에만 있는 경우 일부 DOM 사용 가능 이벤트에서 실행되도록 트리거를 설정

    데이터 영역 변수 생성
        세팅 : 데이터 영역 변수 'customSearchInput' 생성
        작동방식 : 'customSearchInput'은 HTML태그를 통해 수집된 정보 중 사용자 검색어나 자동 맞춤 검색어를 추출하여 변수에 저장한다.

    GA4 이벤트 태그 구현
        세팅1 : 태그 생성 시, 태그 내 이벤트 이름은 정확히 view_search_results로 지정한다.
        세팅2 : 이벤트 매개변수는 'search_term'을 입력하며, 매개변수의 값은 위에서 생성한 'customSearchInput'를 삽입한다
        작동방식 : 트리거로 인해 Fired된 GTM태그는 'view_search_results' 이벤트와 매개변수 'searchTerm'의 값(customSearchInput)을 GA4로 전달

    사용자 정의 이벤트 트리거 생성(customSearch)
        세팅 : 트리거 생성 시, 트리거 내 이벤트 이름은 정확히 customSearch로 지정한다.
        작동방식 : 모든 맞춤 이벤트에 대해 사용자가 검색 이벤트를 발생 시(자동 완성 검색 포함) 트리거 실행된다
  • 사용자 정의 HTML 태그(리스너) 삽입 시 유의사항

  • 사용자 정의 HTML 태그(리스너) 구현 시 주의사항   Key Precautions

    예시 코드는 기본적으로 작동하지 않기에 수정이 필요하다.
        최소한 var searchField 코드가 있는 6행은 반드시 수정 필요
    자세한 안내 가이드는 아래 링크를 통해 확인한다.
        https://www.simoahava.com/analytics/track-autocomplete-search-google-tag-manager/

  • 검색결과 동적 할당   Dynamic Value Allocation

    각 매개변수는 검색 완료페이지에서 해당하는 값이 할당되어야 함

    검색 이벤트 발생시, 각각의 검색 정보가 이벤트 내 매개변수에 동적으로 할당되도록 코드 구현 필요


  • 필수 매개변수   Required parameters

    매개변수는 몇 개를 선택하여 사용 가능하지만, 필수 매개변수는 반드시 구현되어야 함

    searchTerm은 필수 매개변수

GTM - GA4 검색어 추적 구현(사용자 정의 HTML 태그) > Setting Process

Process 1

1. 사용자 정의 HTML 태그 구성

HTML 구성 단계

  • Process
    01
    사용자 정의 HTML 태그 구성 
     1번 : [태그] 탭 선택
     2번 : [새로만들기]를 클릭하면 오른쪽에서 태그 생성창이 열림
    사용자 정의 HTML 태그 구성 
     3번 : 태그 이름 작성. 맞춤 HTML태그를 구현하는 것이므로 '[GA4] Custom HTML_search'로 이름 지정
     4번 : [태그 구성]을 클릭하면 다시 한번 오른쪽에서 태그 목록이 열림
  • Process
    02
    사용자 정의 HTML 태그 구성 
     1번 : [맞춤설정]에서 [맞춤 HTML]태그 선택한다.
    사용자 정의 HTML 태그 구성 
     2번 : 태그 유형에는 [맞춤 HTML]태그가 제대로 선택됬는지 확인한다.
     3번 : 상단 코드를 참고하여 작업하고자 하는 페이지의 자동 완성 검색어를 추적할 수 있는 코드를 작성하여 삽입한다.
        예시 코드는 기본적으로 작동하지 않기에 수정이 필요하다.
        최소한 var searchField 코드가 있는 6행은 반드시 수정 필요하다.

Process 2

1. 사용자 정의 HTML 트리거 구성

HTML 구성 단계

  • Process
    03
    사용자 정의 HTML 트리거 구성 
     1번 : [트리거]를 클릭하면 오른쪽에 트리거 유형을 선택할 수 있는 창이 열린다.
    사용자 정의 HTML 트리거 구성 
     2번 : 트리거 선택에서 [+]를 클릭
  • Process
    04
    사용자 정의 HTML 트리거 구성 
     1번 : 트리거 이름을 입력. [DOM 사용 가능]이라는 새 트리거를 만들 것이기에, 이름은 '[GA4] DOM Ready'로 지정하였다.
     2번 : [트리거 구성]을 클릭한다
    사용자 정의 HTML 트리거 구성 
     2번 : 트리거 유형에서 [DOM 사용 가능]을 선택
  • Process
    05
    사용자 정의 HTML 트리거 구성 
     1번 : [DOM 사용 가능]으로 올바르게 세팅되었는지 확인한다.
        검색 필드가 모든 페이지에 있는 경우 모든 DOM 사용 가능 이벤트에서 실행되도록 트리거를 설정
        검색 필드가 특정 페이지에만 있는 경우 일부 DOM 사용 가능 이벤트에서 실행되도록 트리거를 설정
     2번 : [저장]버튼을 클릭한다.
    사용자 정의 HTML 트리거 구성 
     3번 : 네모칸과 같이 세팅이 완료되었다면 정상적으로 진행 된 것이다.
     4번 : [저장]버튼을 클릭한다.
  • Process
    06
    사용자 정의 HTML 트리거 구성 
     1번 : 맞춤 HTML이 태그 목록에 정상적으로 추가되었다.

Process 2

2. 변수 구성

변수 구성 단계

  • Process
    07
    변수 구성 
     1번 : [변수]를 클릭
     2번 : [새로 만들기]를 클릭
    변수 구성 
     3번 : 변수 이름을 지정. [customSearchInput] 에 대한 변수이므로 [GA4]customSearchInput로 이름 지정
     4번 : [변수 구성]을 클릭
  • Process
    08
    변수 구성 
     1번 : 변수 유형에서 [데이터 영역 변수]를 선택한다.
    변수 구성 
     2번 : 태그 유형에는 [데이터 영역 변수]가 제대로 선택됬는지 확인한다.
     3번 : 중요! 변수 이름은 정확히 customSearchInput (대소문자 구분)이어야 한다.
  • Process
    09
    변수 구성 
     1번 : 네모칸과 같이 세팅이 완료되었다면 정상적으로 진행 된 것이다.
    변수 구성 
     2번 : 올바르게 세팅이 되었다면 저장 후 변수 목록에 노출 됨

Process 3

3. 태그 구성

태그 구성 단계

  • Process
    10
    태그 구성 
     1번 : 태그 탭 선택
     2번 : 새로만들기를 클릭하면 오른쪽에서 태그 생성창이 열림
    태그 구성 
     3번 : 태그 이름 작성. 구글에서 지정한 공식 이벤트 이름인 view_search_results로 지정. 해당 방법은 view_search_results를 추적하는 1번째 방법이므로 (2)을 부여
     4번 : [태그 구성]을 클릭하면 다시 한번 오른쪽에서 태그 목록이 열림
  • Process
    11
    태그 구성 
     1번 : [Google 애널리틱스 : GA4 이벤트] 선택
    태그 구성 
     2번 : 태그 유형에는 [Google 애널리틱스 : GA4 이벤트]가 제대로 선택됬는지 확인
     3번 : 구성 태그에는 '[GA4]측정 ID태그'를 선택(GA4의 측정ID를 불러오는 태그)
     4번 : 이벤트 이름에는 'view_search_results'를 입력(소문자로만). 구글에서 지정해준 이벤트 이름
     5번 : [이벤트 매개변수]를 클릭해서 추가 메뉴를 확장
     6번 : [매개변수 이름]에는 'search_term'를 입력(소문자로만). 구글에서 지정해준 매개변수 이름
     7번 : [값]에서 우측 '+' 버튼을 클릭하여 세팅창으로 이동
  • Process
    12
    태그 구성 
     1번 : 조금전 생성한 맞춤 데이터 변수인 '[GA4] customSearchInput' 선택
    태그 구성 
     2번 : 상단 이미지와 같이 세팅되면 올바르게 진행된 것

Process 3

4. 트리거 구성

트리거 구성 단계

  • Process
    13
    트리거 구성 
     1번 : [트리거]를 클릭하면 오른족에 트리거 유형을 선택할 수 있는 창이 열림
    트리거 구성 
     2번 : 트리거 선택에서 [+]를 클릭
  • Process
    14
    트리거 구성 
     1번 : 트리거 이름을 입력. view_search_results 태그와 일관성을 유지하기 위해 [GA4] Search Trigger로 입력함
     단, 해당 방법은 Search Trigger를 추적하는 2번째 방법이므로 (2)을 부여
     2번 : [트리거 구성]을 클릭한다
    트리거 구성 
     3번 : 트리거 유형에서 [맞춤 이벤트]를 선택
  • Process
    15
    트리거 구성 
     1번 : [맞춤 이벤트]로 올바르게 세팅되었는지 확인
     2번 : 이벤트 이름은 정확히 [customSearch] 여야한다.
     3번 : 이 트리거는 [모든 맞춤 이벤트]에서 실행 되어야만한다.
     4번 : [저장] 선택
    트리거 구성 
     5번 : 네모칸과 같이 세팅이 완료되었다면 정상적으로 진행 된 것
     6번 : [저장] 버튼을 선택하고 완료
  • Process
    16
    트리거 구성 
     1번 : [태그] 및 [트리거]가 정상적으로 저장되고 있다.
    트리거 구성 
     2번 : 맞춤 HTML이 태그가 목록에 정상적으로 추가되었다.

     3번 : [게시]를 클릭하면 모든 과정이 완료되고 가상 태그가 웹 사이트에 구현
