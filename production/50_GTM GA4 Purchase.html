예시 코드 설명

  • 전자상거래 구매추적 코드 예시
  • 매개변수 이름
  • 값
  • items
  • 상품 정보
  • transaction_id
  • 주문 번호
  • affiliation
  • 가맹, 소속, 홈페이지 이름
  • value
  • 최종 결제 금액
  • tax
  • 세금
  • shipping
  • 배송비
  • currency
  • 화폐 구분(KRW)
  • coupon
  • 쿠폰 이름
  • 코드 삽입 시 유의사항

  • 거래별 동적 가치 할당   Dynamic Value Allocation

    각 매개변수는 결제 완료페이지에서 해당하는 값이 할당되어야 함

    거래발생시, 각각의 거래 정보가 이벤트 내 매개변수에 동적으로 할당되도록 코드 구현 필요


  • 단일 거래정보, 복수 제품정보   Single transaction information, multiple product information

    1개의 거래에 제품이 여러 개인 경우, items코드는 여러번 호출되어야 함

    좌측 예시코드에 한글로 써 둔 결과값은 실제 거래 발생 시 거래에서 해당하는 정보를 가져와야 함


  • 필수 매개변수   Required parameters

    매개변수는 몇 개를 선택하여 사용 가능하지만, 필수 매개변수는 반드시 구현되어야 함

    transaction_id, value, items_(name, id, price, variant, quantity)는 필수 매개변수


  • GTM - GA4 Purchase 개발 가이드 문서   Guide Document

    https://developers.google.com/tag-manager/ecommerce-ga4#measure_purchases


GTM - GA4 전자상거래 구매 추적 구현 > Setting Process

Process 1

1. 변수 구성

변수 구성 단계

  • Process
    01
    변수 구성 
     1번 : [변수]를 클릭
     2번 : [새로 만들기]를 클릭
    변수 구성 
     3번 : 변수 이름을 지정. 구매 관련 변수는 일반적으로 Ecommerce.items를 많이 사용
     4번 : [변수 구성]을 클릭
  • Process
    02
    변수 구성 
     1번 : 변수 유형에서 [데이터 영역 변수]를 선택
    변수 구성 
     2번 : 데이터 영역 변수 이름에 [ecommerce.purchase.items]를 입력.
     3번 : [저장]을 클릭
  • Process
    03
  • 변수 이름
  • 데이터 영역 변수 이름
  • 버전
  • [GA4] Ecommerce.transaction_id
  • ecommerce.purchase.transaction_id
  • 버전2
  • [GA4] Ecommerce.affiliation
  • ecommerce.purchase.affiliation
  • 버전2
  • [GA4] Ecommerce.value
  • ecommerce.purchase.value
  • 버전2
  • [GA4] Ecommerce.tax
  • ecommerce.purchase.tax
  • 버전2
  • [GA4] Ecommerce.shipping
  • ecommerce.purchase.shipping
  • 버전2
  • [GA4] Ecommerce.currency
  • ecommerce.purchase.currency
  • 버전2
  • [GA4] Ecommerce.coupon
  • ecommerce.purchase.coupon
  • 버전2
    변수 구성 
     우측표 참고 : 같은 방법으로 데이터 영역 변수를 우측 표를 참고하여 추가로 만듬
     1번 : 변수를 모두 만든 모습. 실제로 이 변수들은 '거래 완료 페이지'에서 호출되는 매개변수 개수만큼 매칭해서 만들어야 함.
    상단에서 예시로 든 코드에서는 8개의 매개변수를 예시로 했기에 8개의 변수를 생성함
    데이터 영역 변수 
     변수이름 / 데이터 영역 변수 이름 / 버전을 잘 선택하여 세팅

Process 2

2. 태그 구성

태그 구성 단계

  • Process
    04
    태그 구성 
     1번 : 태그 탭 선택
     2번 : 새로만들기를 클릭하면 오른쪽에서 태그 생성창이 열림
    태그 구성 
     3번 : 태그 이름 작성
     4번 : [태그 구성]을 클릭하면 다시 한번 오른쪽에서 태그 목록이 열림
  • Process
    05
    태그 구성 
     1번 : Google 애널리틱스 : GA4 이벤트 선택
    태그 구성 
     2번 : 구성 태그에는 '[태그]구글 애널리틱스4_페이지 뷰'를 선택
     3번 : 이벤트 이름에는 'purchase'를 입력(소문자로만). 구글에서 지정해준 이벤트 이름
     4번 : [이벤트 매개변수]를 클릭해서 추가 메뉴를 확장
  • Process
    06
    태그 구성 
     1번 : 부족한 행은 [행추가]를 클릭해서 추가할 수 있음
     2번 : 매개변수 이름에 첫 번째로 [items]를 입력
     3번 : [+]를 클릭해서 변수 선택 창을 활성화
    태그 구성 
     4번 : 미리 생성해둔 데이터 영역 변수 [GA4] Ecommerce.items를 선택
  • Process
    07
    태그 구성 
     1번 : 같은 방법으로 나머지 항목들도 모두 채움
     GA대시보드 : 여기서 작성하는 매개변수들이 실제 거래 완료 페이지에서 스크립트가 호출될 때 GA4 대시보드에 표시 됨
    태그 구성 
     2번 : 모든 세팅을 완료하면 위와 같은 화면이 출력
     트리거 구현 : 아래 프로세스를 진행하여 트리거를 구현

Process 3

3. 트리거 구성

트리거 구성 단계

  • Process
    06
    트리거 구성 
     1번 : [트리거]를 클릭하면 오른족에 트리거 유형을 선택할 수 있는 창이 열림
    트리거 구성 
     2번 : 트리거 선택에서 [+]를 클릭
  • Process
    07
    트리거 구성 
     1번 : 트리거 이름을 입력. Purchase태그와 일관성을 유지하기 위해 [GA4] Purchase Trigger로 입력함
     2번 : [트리거 구성]을 클릭한다
    트리거 구성 
     3번 : 트리거 유형에서 [맞춤 이벤트]를 선택
  • Process
    07
    트리거 구성 
     1번 : 이벤트 이름에 [gtm.dom]을 입력
     2번 : 이 트리거는 [일부 맞춤 이벤트]에만 실행되어야만 함
     3번 : '기본 제공 변수'를 클릭하고 [Page URL]을 선택
     4번 : '정규 표현식과 일치'를 선택
     5번 : 값으로는 웹사이트의 실제 거래 완료 페이지의 URL을 넣음
    가이드에서는 예시로 https://example/order/complete/015878932554 형태의 거래 완료 페이지를 갖는 웹사이트를 추적하는 정규식을 사용함. 여기에서 난수가 주문번호를 뜻함
     6번 : [저장]을 클릭
    트리거 구성 
    네모칸과 같이 세팅이 완료되었다면 정상적으로 진행 된 것
     7번 : 저장 버튼을 선택하고 완료

Process 4

4. 예외 설정

예외 설정 단계

  • Process
    04
    예외 설정 
     작업 취지 : 거래 완료 페이지에서 '페이지 뷰' 데이터가 중복으로 발생되는 것을 막는 작업을 해야 함
     1번 : 기존에 생성해둔 '[GA4] 측정 ID 태그'를 선택
    태그 구성 
     2번 : 트리거 영역을 선택
     3번 : [예외 추가]를 클릭
  • Process
    05
    태그 구성 
     1번 : 트리거 선택에서 '[트리거]Event'를 클릭
    태그 구성 
     2번 : 기존 '[GA4] 측정 ID 태그'는 모든 페이지가 로드될 때마다 트리거 되는 설정이었음. 이번에 'purchase'태그를 추가하면서 '거래 완료 페이지'에서 트리거 되는 '페이지 뷰' 유형을 추가했기 때문에 중복 수집을 방지하기 위해 기존 태그가 '거래 완료 페이지'에서 트리거 되지 못하도록 예외 설정을 하는 것
     3번 : [저장]을 클릭

Process 5

5. 태그 최종 게시(웹사이트 배포)

작업 완료된 태그 최종 게시

  • Process
    08
    최종 게시 전 테스트 및 디버깅  
     조금 전 생성한 태그가 작업공간의 변경사항에 추가 된 것을 확인
     이상 없을 경우 미리보기 클릭
    최종 게시 전 테스트 및 디버깅  
     구글 태그 매니저 컨테이너 스크립트가 삽입된 홈페이지 URL을 삽입
     [connect]를 클릭하면 홈페이지가 열리면서 미리보기 모드와 연결 됨
  • Process
    09
    최종 게시 전 테스트 및 디버깅  
     연결이 완료되면 팝업창이 Connected로 내용이 변경 됨
     Continuned를 클릭하여 테스트 실행
    최종 게시 전 테스트 및 디버깅  
     미리보기 화면에서 [Tags Fired] 영역에 설정해둔 태그들이 보인다면 제대로 작동하는 것
     의도한 대로 조건에 맞게 태그가 Fired 되는지도 체크
  • Process
    10
    태그 배포  
     실제로 웹 사이트에 가상 태그를 배포하는 작업
     구글 태그 매니저 메인 화면에 접속하여 [제출]을 클릭
    태그 배포  
     1번 : 구글 태그 매니저는 협업으로 많이 활용하기에 해당 작업을 누가 했는지 표식을 남김. 대체로 날짜와 작업자 이름 기입
     2번 : 버전 설명에는 배포한 태그명을 작성. 또는 편의대로 설명을 추가해도 무방
     3번 : [게시]를 클릭하면 모든 과정이 완료되고 가상 태그가 웹 사이트에 구현
  • Process
    11
    태그 배포  
     "듣는 사람이 없는 것 처럼 노래를 부르세요" 라는 문구가 등장하면 태그가 올바르게 배포되고 있는 것
    태그 배포  
     배포가 완료되었으며 배포된 버전에 대한 설명 화면이 노출 됨
  • Process
    12
    태그 배포  
     1번 : [게시]한 후에는 작업공간의 변경사항이 초기화
     2번 : 방금 게시한 버전의 번호와 작업자 이메일, 게시 날짜 등이 표시 됨
    태그 배포  
     배포가 완료되었으며 배포된 태그는 태그 탭에 목록형으로 노출
